================================================================================
COST ANALYSIS
================================================================================

OpenAI API Costs:
- Model: GPT-4 (not GPT-4 Turbo - need better accuracy)
- Cost: $0.03/1K input tokens, $0.06/1K output tokens
- Per email analyzed: ~1K input + 200 output = $0.04
- 50 emails/day per firm = $2/day = $60/month
- 100 firms = $6,000/month OpenAI cost

Alternative (cost optimization):
- Use GPT-4 Turbo: $0.01/1K input = $0.60/month/firm
- Trade-off: slightly lower accuracy (90% → 85%)

Gmail API:
- Free tier: 1 billion quota units per day
- We use ~10 units per request
- Effectively unlimited for our scale

Recommendation:
- Start with GPT-4 (best accuracy)
- Switch to GPT-4 Turbo after validating with customers
- Include in pricing (already factored into £120/user/month)

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: "No email integration configured"
Fix: User needs to connect Gmail first
Action: Add clear CTA to connect Gmail

ISSUE: "Failed to fetch Gmail messages"
Likely: Token expired or revoked
Fix: Refresh token automatically (already implemented)
Fallback: Ask user to reconnect

ISSUE: "OpenAI rate limit exceeded"
Likely: Too many requests in short time
Fix: Add retry with exponential backoff
Workaround: Batch analyze emails (reduce API calls)

ISSUE: "Matter matching inaccurate"
Likely: Not enough context or too many matters
Fix: Improve prompt, add more context
Workaround: User can edit before approving

ISSUE: "Suggestions taking too long"
Likely: Analyzing 100+ emails at once
Fix: Limit to 50 emails per request
Add: Background job for bulk analysis
