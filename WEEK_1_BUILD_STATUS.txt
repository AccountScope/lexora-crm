Features implemented:

□ AI-powered time estimation
□ Matter matching with confidence scoring
□ Bulk select and approve workflow
□ Edit suggestions before approval
□ Activity code auto-assignment (UTBMS codes)
□ High/medium/low confidence badges
□ Stats dashboard (total suggestions, hours, confidence)
□ Real-time UI with loading states
□ Error handling and retry logic

How it works:
1. User clicks "AI Suggestions" tab in time tracking
2. Select source (Email, Calendar, or Documents)
3. AI analyzes last 7 days of activity
4. Shows suggestions with:
   - Matched matter (if found)
   - Estimated hours
   - Description
   - Confidence score
   - Activity code
5. User can:
   - Select multiple suggestions
   - Edit hours/description
   - Reject suggestions
   - Bulk approve to create time entries
6. One click → 10-20 time entries logged

3. Matter matching is basic:
   - Currently uses string.includes() matching
   - Need GPT-4 for semantic matching

4. No suggestion history:
   - Should save suggestions to DB
   - Track approved/rejected
   - Use for learning

5. Bulk approve doesn't handle failures:
   - Need transaction support
   - Rollback if one entry fails
   - Better error reporting

6. No rate limiting:
   - Gmail API: 250 quota units per user per second
   - Calendar API: 1,000,000 queries per day
   - Need to handle rate limits gracefully

7. Performance:
   - Analyzing 1000s of emails could be slow
   - Need background job processing
   - Cache suggestions

INTEGRATION REQUIREMENTS
================================================================================

GOOGLE APIS:
1. Enable in Google Cloud Console:
   - Gmail API
   - Google Calendar API
2. OAuth consent screen:
   - Scopes: gmail.readonly, calendar.readonly
3. Create credentials:
   - OAuth 2.0 Client ID (Web application)
   - Add authorized redirect URI
4. Store in .env:
   - GOOGLE_CLIENT_ID
   - GOOGLE_CLIENT_SECRET
   - GOOGLE_REDIRECT_URI

MICROSOFT APIS:
1. Register app in Azure AD:
   - API permissions: Mail.Read, Calendars.Read
2. Create client secret
3. Store in .env:
   - MICROSOFT_CLIENT_ID
   - MICROSOFT_CLIENT_SECRET
   - MICROSOFT_REDIRECT_URI

OPENAI API:
1. Get API key from platform.openai.com
2. Store in .env:
   - OPENAI_API_KEY
3. Estimated cost:
   - 100 suggestions/day × 1,000 tokens × £0.03/1K = £3/day
   - £90/month per firm
   - Factor into pricing (or cap suggestions)

DATABASE CHANGES:
1. Create email_integrations table:
   - user_id, provider, access_token, refresh_token, expires_at
2. Create time_suggestions table:
   - id, user_id, source, source_id, matter_id, hours, description, confidence, status, created_at
3. Add indexes:
   - time_suggestions(user_id, status, created_at)
   - time_suggestions(matter_id)
